01_problem-statement

Toolkits can't scale on manual research

Every integration Composio ships starts with the same question: can an agent actually use this app? Answering it means reading developer docs, checking auth flows, and hunting for MCP support — one app at a time. That process doesn't scale to hundreds of apps, and it goes stale the moment an app ships a new MCP server.

This project builds a repeatable research pipeline instead: an AI agent that reads official documentation, produces a structured verdict for each app, and flags the cases that genuinely need a person to look — so human time goes toward judgment calls, not lookup work.

WHAT DOESN'T SCALE

Manually checking docs, auth methods, and MCP status for hundreds of apps, then re-checking them as MCP ecosystems change weekly.

WHAT THIS BUILDS

A research agent + validation loop that turns "is this app agent-ready?" into a structured, evidenced, re-runnable answer.

AGENT TASKS PER APP

  • Extract authentication methods
  • Identify API surface (REST, GraphQL, SDK, CLI)
  • Detect MCP availability and type
  • Assign a buildability verdict
  • Store evidence URLs for every claim

Output: one validated JSON record per app, conforming to a fixed schema — not free-text notes.

SCOPE & ASSUMPTIONS
  • Verdicts are based on public documentation, not live API testing.
  • Confidence scores reflect evidence quality, not ground truth.
  • 10 categories were chosen to mirror Composio's existing toolkit surface.
  • "Buildable" means an agent could authenticate and call the API — not that a full integration is shipped.

10_limitations

Where to trust this less

A structured dataset can still be wrong in ways schema validation won't catch. These are the honest edges of this research.

  • MCP availability changes rapidly and requires periodic re-verification.
  • Some enterprise APIs can't be fully verified without existing customer access.
  • Public documentation doesn't always reflect production availability.
  • Community MCP servers need additional validation before production use.
11_final-conclusion

A research workflow that scales with the ecosystem

Across 100 applications, most were technically feasible, authentication converged around OAuth2 and API keys, and MCP availability turned out to be the strongest readiness signal in the dataset. The real blockers were commercial access and ecosystems that change faster than documentation does. The next step is a continuous verification loop that keeps MCP and API intelligence current — so the answer to "is this app agent-ready?" stays true after the research is done.
